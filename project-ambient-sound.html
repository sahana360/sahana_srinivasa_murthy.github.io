Overview

Helping the hearing-impaired connect with their environment

According to the most recent Indian census, 63 million people suffer from significant auditory loss. People with Hearing Impairment (PWHI) encounter difficulties in their daily lives even in domestic environments — unable to hear sounds that are essential for carrying out daily routines or recognizing potential dangers.

This project detects and classifies essential sounds in a user's ambiance using a Deep Learning model, sending notifications to their smartphones via a mobile application. The system runs in real time using a Raspberry Pi 3 with a USB microphone, communicating with a cloud-hosted classification model.

The project was awarded Best Project at BNM Institute of Technology and subsequently published in the International Journal of Ambient Systems and Applications (AIRCC, September 2022).

Dataset Sources

Training data was sourced from two repositories: Google AudioSet — a large-scale collection of labeled 10-second audio clips containing hundreds of noisy samples per class, and FreeSound — providing clean, high-clarity audio clips of approximately 10 seconds each.

Data augmentation was applied to the clean dataset using time shift, pitch shift, and time stretch techniques via the Audiomentations and Librosa libraries to increase training diversity and robustness.

Spectrogram Processing

Raw WAV audio files are pre-processed using Short-Time Fourier Transform (STFT), which captures time-localized frequency information — outperforming standard FFT for non-stationary audio signals.

Log-Mel Spectrograms are then computed from the STFT output. This logarithmic mel-scale representation aligns with human auditory perception and serves as the visual input fed into the CNN architecture at a resolution of 224×224 pixels.

Model Comparison

Six architectures evaluated. ResNet50 chosen.

Six pre-trained CNN architectures — originally trained on ImageNet — were fine-tuned on the spectrogram dataset using transfer learning. All models were trained for 15 epochs with the Adam optimizer (lr=0.001) and an 80:10:10 train/val/test split.

ResNet50 and Xception both achieved 100% test accuracy. ResNet50 was selected for deployment due to its significantly lower memory footprint, faster inference time, and easier integration with TensorFlow Lite for cloud hosting.

ResNet50 Architecture

Why ResNet50 works

Skip Connections

Skip connections feed the output of one layer directly to layers further down the network — allowing each layer's output to theoretically depend on all previous layers, not just the preceding one.

Vanishing Gradient Solution

Identity connections protect the network from the vanishing gradient problem during backpropagation, ensuring all 50 layers learn effectively and accuracy does not saturate prematurely.

Bottleneck Design

Each residual block uses 1×1, 3×3, 1×1 convolution stacks — the 1×1 layers reduce then restore dimensions, making the 3×3 layer a computational bottleneck that improves efficiency without loss in performance.

TF Lite Compression

The trained ResNet50 model was compressed to TensorFlow Lite format, enabling lightweight deployment on the PythonAnywhere cloud server for real-time inference.

System Architecture — Three Modules

The system is divided into three interconnected modules that handle data collection, classification, and user notification respectively.

📡
IoT — Sound Collection
A Raspberry Pi 3 with a USB microphone records 5-second ambient audio clips at 48,000 Hz in WAV format using a headless SSH setup via ALSA / arecord. Audio is uploaded to Firebase Storage over Wi-Fi using the pyrebase library, making it available for the server to process on demand.
☁️
Server — Classification
A Flask REST API hosted on PythonAnywhere handles incoming GET requests from the mobile app every 10 seconds. On each call, it fetches audio from Firebase, runs STFT → Log-Mel spectrogram pre-processing with Librosa, feeds the result into the ResNet50 TF Lite model, and returns the classification result in JSON format.
📱
Android App — Notification
Built in Android Studio (Java), the app polls the server every 10 seconds using the Volley library. When a sound is detected, it displays a dynamic card with English text and a pictorial icon, and alerts the user via vibration or torch light. Users choose their preferred notification mode, and duplicate sounds are suppressed until dismissed.
